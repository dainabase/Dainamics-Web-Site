## Copilot, l'IA au cœur de Microsoft 365

Ignite 2025 a confirmé la stratégie de Microsoft : intégrer l'IA dans chaque recoin de la suite Office. Les annonces de Work IQ et Agent 365 marquent une nouvelle étape dans la transformation des outils de productivité.

Pour les PME suisses déjà équipées en Microsoft 365, comprendre ces évolutions est essentiel pour maximiser leur investissement.

## Les annonces clés d'Ignite 2025

### Work IQ : L'intelligence contextuelle

**Concept :**
Work IQ est une couche d'intelligence qui comprend votre travail dans son ensemble, pas juste document par document.

**Fonctionnalités :**
- Analyse automatique de tous vos contenus (emails, documents, réunions)
- Suggestions proactives basées sur votre contexte
- Identification des informations pertinentes avant que vous ne les cherchiez
- Connexions automatiques entre projets et personnes

**Exemple concret :**
```
Vous ouvrez un email d'un client.
Work IQ vous montre automatiquement :
- Le dernier devis envoyé
- Les notes de la dernière réunion
- Les collègues qui ont interagi avec ce client
- Les documents pertinents
```

### Agent 365 : Le contrôle des agents

**Concept :**
Une plateforme centralisée pour déployer, gérer et superviser les agents IA dans l'entreprise.

**Fonctionnalités :**
- Création d'agents sans code (Copilot Studio)
- Gestion des permissions et accès
- Monitoring centralisé
- Bibliothèque d'agents prêts à l'emploi

**Cas d'usage entreprise :**
- Agent RH pour les questions employés
- Agent IT pour le support niveau 1
- Agent commercial pour la qualification de leads
- Agent projet pour le suivi des tâches

### Computer Use : L'automatisation visuelle

**Concept :**
Les agents peuvent maintenant contrôler des applications comme un humain (souris, clavier virtuels).

**Implications :**
- Automatisation d'applications sans API
- Intégration de logiciels legacy
- Macros intelligentes

**Limites actuelles :**
- Lenteur (manipulation visuelle)
- Fragilité (si l'interface change)
- Sécurité (accès étendu)

## Impact sur les workflows Office

**Nouvelles capacités :**

| Rédaction | Suggestion de texte | Rédaction complète avec style adapté |


## Déploiement dans une PME

### Phase 1 : Préparation (1-2 semaines)

**1. Audit des usages actuels**
- Quels outils Office sont utilisés ?
- Par qui et pour quoi ?
- Où sont les gisements de productivité ?

**2. Sélection des pilotes**
Profils idéaux pour commencer :
- Assistants de direction (beaucoup de rédaction)
- Commerciaux (propositions, emails)
- Chefs de projet (rapports, présentations)
- Analystes (Excel intensif)

**3. Préparation des données**
Copilot fonctionne mieux avec :
- Documents bien organisés (SharePoint/OneDrive)
- Emails structurés
- Calendriers à jour

### Phase 2 : Pilote (4-6 semaines)

**Semaine 1-2 : Formation**
- Formation initiale (2-4h)
- Cas d'usage prioritaires
- Bonnes pratiques de prompting

**Semaine 3-4 : Utilisation guidée**
- Support disponible
- Sessions de partage d'expérience
- Collection des feedbacks

**Semaine 5-6 : Évaluation**
- Mesure des gains
- Identification des obstacles
- Ajustement des pratiques

### Phase 3 : Déploiement (progressif)

**Vague 1 :** Utilisateurs intensifs (ceux qui en bénéficieront le plus)
**Vague 2 :** Utilisateurs réguliers
**Vague 3 :** Utilisateurs occasionnels (évaluer si pertinent)

### Phase 4 : Optimisation (continue)

- Partage des meilleures pratiques
- Création de templates et prompts réutilisables
- Formation continue
- Mesure régulière de l'adoption

## Bonnes pratiques

### Pour les utilisateurs

**1. Apprendre à prompter**
```
Mauvais : "Fais un résumé"
Bon : "Résume ce document en 5 points clés pour une présentation à la direction"
```

**2. Itérer**
- Première génération rarement parfaite
- Affiner avec des instructions supplémentaires
- Combiner avec l'expertise humaine

**3. Vérifier**
- Copilot peut se tromper
- Toujours relire avant envoi/publication
- Particulièrement pour les chiffres et faits

### Pour les administrateurs

**1. Gouvernance des données**
- Copilot accède aux données autorisées
- Vérifier les permissions SharePoint/OneDrive
- Attention aux données sensibles

**2. Monitoring de l'adoption**
- Qui utilise vraiment ?
- Pour quels usages ?
- Où sont les blocages ?

**3. Support continu**
- FAQ interne
- Champions Copilot par département
- Sessions de partage régulières

## Limites et précautions

### Ce que Copilot ne fait pas bien (encore)

**1. Données temps réel**
- Pas d'accès à Internet
- Limité aux données de votre tenant

**2. Calculs complexes**
- Peut se tromper sur les chiffres
- Vérification nécessaire

**3. Contexte externe**
- Ne connaît pas vos concurrents
- Pas d'actualités récentes

**4. Tâches très spécifiques**
- Métiers de niche
- Jargon très spécialisé

### Risques à gérer

**Confidentialité :**
- Copilot peut surfacer des infos auxquelles l'utilisateur a accès
- Attention aux permissions trop larges
- Revue régulière des accès

**Dépendance :**
- Ne pas perdre les compétences de base
- Maintenir l'esprit critique
- Équilibre automatisation/expertise

**Coûts :**
- 30 CHF/user/mois s'accumule
- Évaluer régulièrement l'utilisation réelle
- Désactiver pour les non-utilisateurs

## Alternatives et compléments

### Pour les PME sans budget Copilot

**Options gratuites/moins chères :**
- ChatGPT/Claude via navigateur (copier-coller)
- Extensions tierces (moins intégrées)
- Google Workspace + Gemini (alternative complète)

### Compléments à Copilot

**Pour aller plus loin :**
- Power Automate + Copilot Studio (automatisations)
- Azure OpenAI (API pour applications custom)
- Agents spécialisés (tâches spécifiques)

## Conclusion : Pour qui Copilot vaut-il l'investissement ?

### Fortement recommandé

- Rôles à forte composante rédactionnelle
- Analystes manipulant beaucoup de données
- Managers gérant beaucoup de réunions
- Équipes projets collaboratives

### À évaluer

- Rôles techniques spécialisés
- Postes avec peu de travail Office
- Équipes avec processus très structurés

### Probablement pas rentable

- Utilisateurs occasionnels d'Office
- Rôles terrain sans PC
- PME avec stack non-Microsoft
